# Email Announcement

Subject: BountyLens is ready - screen GitHub bounty issues locally

Hi,

I just released BountyLens, a local-first toolkit for developers who review GitHub bounty issues.

It helps you screen an issue before spending time on it or handing it to Codex:

- Detect bounty, reward, funded, sponsor, and known platform signals.
- Extract possible amounts such as `$50`, `USD 100`, `USDC`, `DAI`, or `ICP`.
- Flag fake or weak reward wording.
- Flag likely competing PRs.
- Flag risky categories such as security, auth, payment, crypto, KYC, and database migration.
- Generate CLI reports in Markdown or JSON.
- Use prompt templates for read-only candidate review, pilot analysis, one draft PR attempt, and PR status checks.

What is included:

- Chrome extension.
- Python CLI.
- Prompt pack.
- Example issue and reports.
- Landing page.
- Sales and policy templates.

BountyLens is local-first. It does not read cookies, GitHub tokens, keychains, password managers, or credentials. It does not claim bounties, comment on issues, create PRs automatically, or process KYC/payment/payout workflows.

It also does not guarantee bounty earnings. It is a practical screening tool for deciding whether an issue deserves deeper manual review.
