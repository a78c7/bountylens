# LinkedIn Launch Post

I built BountyLens, a local-first toolkit for developers and indie hackers who review GitHub bounty issues.

The problem: funded issues are easy to misread. A page may mention a reward, but the signal can be stale, weak, already contested by another PR, or tied to risky categories like auth, payment, crypto, KYC, security, or production database work.

BountyLens gives contributors a lightweight preflight step:

- Chrome extension that scans visible GitHub issue text locally.
- Python CLI that produces Markdown or JSON reports.
- Codex prompt pack for read-only candidate review, pilot analysis, one draft PR attempt, and PR status checks.
- Sales-ready docs, examples, and policy templates.

The product does not read cookies, GitHub tokens, keychains, or password managers. It does not claim bounties, comment on issues, create PRs automatically, or process KYC/payment/payout workflows.

It is not a guarantee of bounty earnings. It is a practical screening workflow for deciding whether an issue deserves deeper manual review before implementation.
