# Gumroad Listing

## Product Title

BountyLens - Local GitHub Bounty Issue Scanner

## Short Description

A lightweight Chrome extension, Python CLI, and Codex prompt pack for screening GitHub bounty issues before you spend time on them.

## Long Description

BountyLens helps open source contributors quickly review public GitHub issue pages for bounty signals, possible reward amounts, fake or weak bounty wording, competing PR risk, and high-risk task categories.

Use it before handing a candidate to Codex or another AI agent. The goal is simple: avoid wasting time on fake, risky, already-claimed, or unsuitable bounty issues.

BountyLens is local-first. The extension analyzes visible GitHub issue text in your browser. The CLI analyzes a public URL, local HTML file, or text that you provide. There is no hosted dashboard, no tracking, no GitHub token, and no payout handling.

## FAQ

### Does BountyLens guarantee I will earn bounties?

No. It helps screen candidates and reduce obvious mistakes. It cannot control maintainer decisions or platform rules.

### Does it need my GitHub token?

No. The extension reads visible page text. The CLI can fetch public issue pages or analyze local text.

### Does it support private repositories?

No. The product is designed for public GitHub issue screening and local examples.

### Does it create pull requests?

No. The included prompts can guide a careful workflow, but the extension and CLI do not create PRs.

### Can I customize the rules?

Yes. You can edit the Python and JavaScript keyword lists for your own use.
