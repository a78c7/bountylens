A safer workflow from GitHub issue to draft PR

Local first. Manual confirmation required.
1

GitHub issue

Open a public issue that mentions a bounty, reward, or funded task.

2

BountyLens triage

Check bounty signal, amount, platform, fake wording, and risk flags.

3

Manual confirmation

Confirm platform status, payout rules, and competing PRs yourself.

4

Codex pilot

Run a read-only review before implementation or repository changes.

5

Docker/testability

Check whether the work can be tested locally without private access.

6

Draft PR

Create at most one draft PR after explicit approval.

7

Maintainer review

Let maintainers review. Handle bounty platform steps manually.

BountyLens does not claim rewards, process payments, handle KYC, or guarantee earnings.
