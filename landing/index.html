Problem

Funded issues are easy to misread.

Reward text can be stale, fake, below platform minimums, already claimed, or tied to risky work that should not be delegated to an AI agent.

Before coding, contributors need a fast way to ask: is this actually a bounty, is it safe, is it still open, and is it worth a deeper review?

Solution

A local preflight check for bounty candidates.

BountyLens combines a browser panel, CLI report, and Codex prompts so you can move from quick screen to manual confirmation to careful draft PR workflow.

Chrome extension

Scans visible GitHub issue text and displays bounty signal, possible amount, platform, risk flags, and recommendation.

Python CLI

Analyze a public issue URL, local HTML file, or raw text. Output Markdown or JSON with no third-party dependencies.

Prompt pack

Use structured Codex goals for read-only review, candidate pilot, one draft PR attempt, and PR status checks.

Safety filters

Flags security, auth, payment, crypto, KYC, database migration, external environment, fake bounty, and competing PR signals.

Safety-first workflow

Screen first. Confirm manually. Then let Codex help.

  1. Open a GitHub issue page.
  2. Read the BountyLens panel.
  3. Confirm bounty platform, payout rules, and competing PRs manually.
  4. Run a read-only Codex candidate review.
  5. Check Docker, tests, and local feasibility.
  6. Create one draft PR only after explicit approval.

FAQ

Common questions

Does BountyLens guarantee bounty earnings?

No. It is a screening workflow. Maintainers and bounty platforms decide outcomes.

Does it send data to a server?

No. The extension analyzes visible page text locally. The landing page loads no tracking scripts.

Does it need a GitHub token?

No. The CLI can read public URLs or user-provided local files and text.
